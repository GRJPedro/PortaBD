Análisis de Base de Datos

Estimación del Tamaño de Base de Datos

  • Se identifican los requisitos de almacenamiento, como determinar tipos y cantidad de datos
  • Se estima el crecimiento de datos proyectando el aumento de datos en el tiempo
  • Se calcula el tamaño inicial considerando factores, como índices, vistas y espacios de almacenamiento
  • Se consideran factores de escala, como crecimiento esperado, tráfico y usuarios

Selección del Servidor

  • Se identifican los requisitos de procesamiento, como determinar el tipo de CPU, memoria y almacenamiento
  • Se evaluan las opciones de infraestructura, como servidores físicos, virtuales, cloud o hibridos, en esta evaluación se determina el uso de servidores de bases de datos geoespaciales
  • Se consideran factores de escalabilidad, como el creciemiento esperado y capacidad de expansión
  • Se evaluan costos y presupuestos

Selección del Manejador de Base de Datos

  • Se identifican los requisitos de gestión, como determinar necesidades de seguridad, backup y recuperación
  • Se evaluan las opciones de manejadores como MySQL, PostgreSQL, MongoDB, Oracle, etc.
  • Se consideran factores de compatibilidad como el Sistema Operativo y lenguajes de programación
  • Se evaluan costos y soporte

Herramientas y Técnicas

  • Se analizan los requisitos funcionales y no funcionales
  • Se hace un modelado de datos (Entidad-Relación, UML)
  • Se calcula el tamaño y creciemiento de datos
  • Se evaluan las herramientas de gestión de bases de datos

Bases de Datos Relacionales

Estas son las principales acciones que realizo en las Bases de Datos Relacionales:

  • Identificar entidades, atributos y llaves
  • Identificación de relaciones y cardinalidad
  • Diagramación (Modelo Entidad-Relación)
  • Normalización de Base de Datos
  • Refinar el Diseño, comprobación y medición

Diagramación en Bases de Datos Relacionales

Identificar las Entidades, Atributos

  • Identifico las entidades principales del sistema
  • Las entidades las convierto en tablas
  • Para cada entidad defino sus atributos (campos o columnas)

Establecer Relaciones y Cardinalidad

  • Identifico la interacción o asociación entre entidades, definiendo si son relaciones fuertes, debiles o recursivas
  • Identifico la cardinalidad, con el número de entidades que pueden ser asociadas con otras entidades, ya sea uno a uno, uno a muchos y muchos a muchos

Crear el Modelo Entidad-Relación (MER)

  • Se realiza un diagrama de flujo representando la relación entre las entidades y como participan en el proceso del negocio, tanto dentro como fuera del sistema
  • Para la diagramación utilizo herramientas como SmartDraw y LucidChart

Normalizar y Definir Llaves

  • Se eliminan datos redundantes y se crean nuevas tablas
  • Se establecen llaves primarias paa cada tabla
  • Se establecen llaves foraneas para relacionar las tablas

Refinar el Diseño

  • Se verifica la consistencia y coherencia del diseño
  • Se revisa que el diseño sea escalable y flexible

Bases de Datos Geoespaciales

Las Bases de Datos Geoespaciales (BDG) son sistemas de información que almacenan, gestionan y analizan datos georreferenciados. Permiten la integración de datos espaciales y no espaciales para realizar consultas y análisis geográficos. Estas son algunos de mis productos con Bases de Datos Geoespaciales, así como Servidores Geográficos:

  • Análisis de Bases de Datos Geoespacial
  • Selección del Tipo de Base de Datos Geoespacial
  • Herramientas y Tecnologías
  • Aplicaciones de Bases de Datos Geoespaciales

Bases de Datos Geoespaciales

Análisis de Base de Datos Geoespacial

  • Se analiza la información geográfica y no geográfica que será almacenada, así como la delimitación espacial (extends) y la proyección y datum que llevara
  • Se determinan el tipo de datos que se almacenaran y serán compartidos con servidores y Sistemas de Información Geográfica como (shapefile, KML, GeoJason, .tab)
  • Se determinan las herramientas y los algoritmos que se usarán en los análisis que se llevaran a cabo en la Base de Datos Geoespacial
  • Se definen las herramientas de consulta y visualización de los datos y mapas

Selección de la Base de Datos Geoespacial

  • Se identifican el tipo de dato geografico como información Vectorial, información Raster (imágenes de satélite o modelos digitales de elevación) o información red (análisis de redes)
  • Se determina la base de datos geoespacial óptima para el tipo de dato geografico, seguridad factores de escalabilidad, como el creciemiento esperado y capacidad de expansión
  • Se evaluan costos y presupuestos

Herramientas y Tecnologías

  • PostgreSQL con su extensión PostGIS
  • Geodatabase de ESRI
  • GeoServer
  • MongoDB
  • QGIS
  • ArcGIS

Temas donde aplico las Bases de Datos Geoespaciales

  • Planificación urbana y territorial
  • Gestión de recursos naturales
  • Logística y transporte
  • Emergencias y riesgos
  • Salud pública y epidemiología
  • Análisis de mercado y comercio

Gestores de Base de Datos

SQL

  • Estos son los principales SGBD que uso en Modelos de Datos Relacionales:
    • PostgreSQL
    • MySQL
  • Estos debido a:
    • Su soporte en transacciones
    • Seguridad y autenticación
    • Escalabilidad y rendimiento
    • Con la extensión PostGIS puedo trabajar con información geográfica

No SQL

  • El SGBD que uso para manejar datos No SQL es MongoDB debido a:
    • No relacional por lo cual trabajo con colecciones y documentos
    • Se puede trabajar con información Geoespacial, creando ínidices geoespaciales y un modelo de datos optimizado
